## 主要观点

2026 年大模型效果真正拉开差距的地方，不再是预训练本身，而在后训练、评测、奖励、Agent 训练、蒸馏等后半段训练栈。模型变强通常是权重、训练链路和部署决策共同作用的结果，不只是参数规模。InstructGPT 证明：1.3B 参数做过对齐的模型，在人类偏好评测里能赢过 175B 的 GPT-3。

## 关键细节

- **九阶段训练流水线**：原始数据 → 数据工程 → 系统配方 → 预训练 → 后训练 → 评测/奖励 → Agent harness → 蒸馏/专用化 → 部署
- **两条反馈回路**：生产流量回到数据工程；离线评测结果回到预训练
- **规模定律实践**：Chinchilla 最优点 8B 参数约 200B tokens，但 Llama3 8B 用了 15T tokens（超出约 75 倍），过训练换取更高能力密度
- **后训练四阶段流水线**（DeepSeek-R1）：冷启动 SFT → 推理 RL (GRPO) → 拒绝采样微调 → 对齐 RL
- **GRPO vs PPO**：GRPO 不需要独立价值网络，用组内排名替代绝对价值估计，工程更简洁
- **ORM vs PRM**：ORM 只给最终答案打分（信号稀疏）；PRM 给中间步骤打分（信号密集，成本高数倍）
- **Agent 训练关键**：harness 稳定性 > 模型训练；环境质量 > 数据多样性
- **Meta-Harness**：同一底模只改 harness 可拉出 6x 性能差距；在线文本分类比 ACE 高 7.7 点，context token 用量压到 1/4
- **蒸馏价值**：大模型先通过 RL 把推理能力练出来，再迁给更小 dense 模型；能力解耦是关键原因

## 方法 / 进路

文章按训练链路顺序展开，从预训练到部署，重点放在后半段训练栈如何影响最终上线效果：

1. **预训练**：打地基，决定知识范围、泛化潜力；tokenizer、context window、多模态取舍在训练阶段就定下来
2. **数据工程**：完整漏斗处理（文本抽取 → 语言识别 → 质量过滤 → PII → 安全 → 去重）；数据配比决定能力分布
3. **系统配方**：分布式系统问题而非单机深度学习；MoE、muP、WSD learning rate、长上下文约束
4. **后训练**：多阶段流水线；SFT 学风格，RLHF/DPO/RFT 接偏好反馈
5. **评测与奖励**：eval 决定测什么，grader 决定分数，reward 决定优化方向；verified rewards 替代人工偏好
6. **Agent 训练**：优化对象从答案 → 轨迹 → harness program；PARL 只训练 orchestrator
7. **蒸馏部署**：前沿模型价值 = 训练数据来源；发布版本是产品决策，不是训练曲线最右点

## 结果 / 证据

- InstructGPT：1.3B 对齐模型赢过 175B GPT-3（参数差两个数量级）
- DeepSeek-V3：预训练 14.8T tokens，约 2.788M H800 GPU hours，无 irrecoverable loss spike，无 rollback
- Meta-Harness：在线文本分类比 ACE 高 7.7 点，context token 用量压到 1/4；检索增强数学推理 200 道 IMO-level 题上，5 个 held-out 模型平均涨 4.7 点
- DeepSeek-R1 蒸馏：RL 后大模型轨迹让 1.5B 到 70B dense 模型都获得明显收益

## 值得保留的引用

> "模型怎么说话、怎么听指令、怎么推理、怎么用工具，这些都不是多喂一点互联网文本就能自然长出来的。"

> "预训练更像是给模型能力打地基，决定知识范围、泛化潜力和模式归纳能力，也决定后训练有没有可以利用的空间。但听不听指令、配不配合用户、关键任务跑起来稳不稳，这些预训练都是管不到的。"

> "到了 Agent 阶段，reward design 还会继续拆细，最终结果只是其中一项，另外还要单独度量过程质量、上下文管理和反作弊约束。"

> "今天发布的模型只是一个快照，链路和 harness program 才是持续在跑的产品。"
