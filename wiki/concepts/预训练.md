# 预训练

## 定义

预训练是大模型训练链路的起点，通过在大规模文本语料上进行下一个 token 预测，将语言分布、知识和模式压缩进模型参数，为后续能力激活预留空间。

## 为什么重要

预训练决定模型的知识覆盖范围、表示质量和规模效率，是模型能力的地基。没有预训练就没有语言建模能力，也没有后训练能力迁移的空间。但预训练管不了指令遵循、用户配合度、关键任务稳定性，这些需要后训练解决。

## 工作原理

1. **下一个 token 预测**：训练目标是预测序列中下一个 token，形式上简单但内涵复杂
2. **知识压缩**：把大规模文本里的知识和模式压进参数
3. **能力预留**：给后训练的能力激活留出空间
4. **规模定律**：参数量、训练 token 数和计算预算之间有配比关系

```
训练决策问题：
- 多堆参数还是多喂数据？
- 当前模型是能力不够还是欠训练？
- 有限 GPU 预算下什么配比更值？
```

## 关键属性 / 权衡

- **Chinchilla 最优点**：8B 参数约 200B tokens
- **过训练配方**：Llama3 8B 用 15T tokens（超出约 75 倍），换取更高能力密度
- **总 FLOP**：比参数量更能预测质量
- **Tokenizer 影响**：词表大小、分词策略影响序列长度、推理成本、多语言能力
  - Llama2 词表 32K → Llama3 扩到 128K，序列长度压缩约 15%
- **架构取舍**：context window、多模态、单卡可运行等在训练阶段就定下来
