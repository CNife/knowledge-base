# Agent 记忆系统

## 定义

Agent 记忆系统是跨会话状态保持的机制，按实际要解决的问题分为四种类型：工作记忆、程序性记忆、情景记忆和语义记忆。Agent 不具备原生的时间连续性，会话结束后上下文清空，记忆层必须单独设计。

## 为什么重要

对 Agent 来说记忆是一层基础设施，不是可以事后补上的能力。没有记忆层，Agent 无法在长任务中保持连贯，也无法积累跨会话的经验。[[agent-loop]] 只持有短期状态（messages[]），记忆系统负责长期状态。

## 工作原理

四种记忆类型：

| 类型       | 存储位置          | 内容                    | 生命周期   | 注入方式       |
|-----------|------------------|------------------------|----------|--------------|
| 工作记忆   | 上下文窗口        | 当前任务所需的最小信息     | 会话级    | 直接进 messages[] |
| 程序性记忆 | Skills 文件       | 怎么做某件事，操作流程     | 持久化    | 按需加载       |
| 情景记忆   | JSONL 会话历史    | 发生了什么              | 持久化    | 检索后注入     |
| 语义记忆   | MEMORY.md        | Agent 认为重要的事实      | 持久化    | 每次启动注入   |

MEMORY.md 与 Skills 协作：
- `memory/YYYY-MM-DD.md`：追加写日志，保留原始细节
- `MEMORY.md`：精选事实，Agent 主动维护
- `memory_search`：70% 向量相似度 + 30% 关键词权重的混合检索

ChatGPT 四层记忆（产品实现）：

| 层                  | 内容                        | 持久化   |
|--------------------|----------------------------|---------|
| Session Metadata   | 设备、地点、使用模式          | 否，会话级 |
| User Memory        | 约 33 条关键偏好事实         | 是，每次注入 |
| Conversation Summary| 约 15 个最近对话的轻量摘要    | 是，摘要预生成 |
| Current Session    | 当前对话滑动窗口             | 否      |

记忆整合触发与回退：
- 触发阈值：`tokenUsage / maxTokens >= 0.5`
- 成功路径：摘要追加到 MEMORY.md，更新 lastConsolidatedIndex
- 失败路径：原始消息写入 archive/，保留完整历史

## 关键属性 / 权衡

- **MEMORY.md 容量**：ChatGPT 约 33 条关键偏好事实
- **整合触发阈值**：50% token 使用率
- **检索策略**：70% 向量相似度 + 30% 关键词权重
- **向量检索引入时机**：规模超过几千条时再考虑，结构化 Markdown + 关键词搜索已足够
