# 授权码授权

## 定义

授权码（authorization code）是 OAuth 2.0 四种授权方式中的一种，适用于有后端的 Web 应用。客户端通过重定向用户到授权服务器，获取授权码，再用授权码向授权服务器请求令牌。

## 为什么重要

授权码方式是最完整、最安全的 OAuth 2.0 授权流程，适用于有后端的 Web 应用。它通过两步交换（先获取授权码，再换取令牌）确保令牌不会暴露在用户代理（浏览器）中 [S2]。

## 工作原理

授权码授权流程 [S2][S3]：

1. **用户跳转到授权服务器**
   - A 网站让用户跳转到 GitHub
   - URL 包含 `client_id` 和 `redirect_uri` 参数

2. **用户登录并授权**
   - GitHub 要求用户登录，确保是本人在操作
   - GitHub 询问用户是否同意授权

3. **获取授权码**
   - 用户同意授权后，GitHub 重定向回 `redirect_uri`
   - URL 带有授权码参数：`code=859310e7cecc9196f4af`

4. **用授权码换取令牌**
   - 后端使用授权码向 GitHub 请求令牌
   - 需要提供三个参数：`client_id`、`client_secret`、`code`
   - GitHub 返回 JSON 数据，包含 `access_token`

5. **使用令牌访问 API**
   - 后端使用令牌向 GitHub API 请求用户数据
   - HTTP 头信息必须带上令牌：`Authorization: token accessToken`

## 关键属性 / 权衡

- **安全性高**：令牌不经过用户代理（浏览器），只在后端传输
- **适用于**：有后端的 Web 应用
- **流程完整**：包含授权码和令牌两步交换
- **需要客户端密钥**：后端必须安全存储 `client_secret`
