# OAuth 2.0

## 定义

OAuth 2.0 是一种授权机制，通过在客户端和资源服务器之间引入授权层，分离客户端和资源所有者的角色，使资源所有者能够授权第三方应用有限、可控、可撤销地访问其数据，而无需共享密码。

## 为什么重要

OAuth 2.0 是目前最流行的授权机制，用来授权第三方应用获取用户数据。它解决了第三方应用访问用户数据时的安全问题：用户不必向第三方透露密码，同时授权是有限、可控、可撤销的。[[令牌]] 作为核心凭证机制，代替密码供第三方应用使用 [S1][S2]。

## 工作原理

OAuth 2.0 引入一个授权层，用来分离两种不同的角色：客户端和资源所有者。资源所有者同意以后，资源服务器可以向客户端颁发令牌。客户端通过令牌去请求数据 [S2]。

用"快递员进小区"类比 [S1]：
- 居民小区 → 储存用户数据的网络服务
- 快递员 → 第三方应用
- 屋主 → 用户本人（资源所有者）
- 门禁系统 → OAuth 授权层
- 令牌 → 临时进入凭证

基本流程：
1. 第三方应用请求授权
2. 用户确认授权请求
3. 授权服务器颁发令牌
4. 第三方应用使用令牌访问资源

OAuth 2.0 规定了四种获得令牌的授权方式（authorization grant）[S2]：
- [[授权码授权]] — 适用于有后端的 Web 应用
- [[隐藏式授权]] — 适用于纯前端应用
- [[密码式授权]] — 适用于高度信任的应用
- [[凭证式授权]] — 适用于命令行应用或服务器到服务器通信

## 关键属性 / 权衡

- **安全性**：通过授权层分离角色，用户无需向第三方透露密码
- **可控性**：令牌有权限范围（scope），如只读 vs 读写
- **可撤销性**：资源所有者可随时撤销令牌，立即失效
- **短期有效**：令牌到期自动失效，降低泄漏风险
- **多场景支持**：四种授权方式适配不同应用场景
