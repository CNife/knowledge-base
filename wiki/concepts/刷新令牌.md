# 刷新令牌

## 定义

刷新令牌（refresh token）是 OAuth 2.0 中的一种特殊令牌，用于在访问令牌过期后，无需用户重新授权即可获取新的访问令牌。刷新令牌由授权服务器颁发，客户端妥善保存，在访问令牌失效时使用。

## 为什么重要

刷新令牌解决了访问令牌短期有效带来的用户体验问题。访问令牌有效期短可以降低安全风险，但用户不必频繁重新授权。通过刷新令牌，客户端可以在后台静息获取新的访问令牌，保持长期授权状态 [S2]。

## 工作原理

刷新令牌使用流程 [S2]：

1. **初始授权获取刷新令牌**
   - 用户完成初始授权流程
   - 授权服务器返回访问令牌和刷新令牌

2. **访问令牌过期**
   - 访问令牌到期失效
   - 客户端检测到 API 请求因令牌失效被拒绝

3. **使用刷新令牌获取新访问令牌**
   - 客户端向授权服务器的令牌端点发送请求
   - 提供刷新令牌、`client_id`、`client_secret`
   - 请求类型指定为 `grant_type=refresh_token`

4. **获取新的访问令牌**
   - 授权服务器验证刷新令牌
   - 返回新的访问令牌（可能同时返回新的刷新令牌）

5. **继续使用新令牌访问资源**
   - 客户端使用新的访问令牌继续 API 请求

## 关键属性 / 权衡

- **长期授权**：允许用户在一次授权后长期访问资源
- **安全性**：刷新令牌有效期更长，必须安全存储
- **用户体验**：用户不必频繁重新授权
- **可撤销**：用户可以撤销刷新令牌，立即终止所有访问
- **仅用于后端**：刷新令牌不应暴露在前端或浏览器中
