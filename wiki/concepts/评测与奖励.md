# 评测与奖励

## 定义

评测与奖励是将模型输出转化为训练分数的反馈回路：eval 决定测什么，grader 决定一次输出怎么变成分数，reward 决定模型后面会被往哪里推。它们连起来就是任务定义 → eval → grader → 优化 → rollout → 再评测的循环。

## 为什么重要

只看最终答案，模型会走捷径；打分太粗，噪声被强化学习放大；榜单涨了，真实任务未必跟着好。很多时候用户以为在看 base model 差距，其实差距出在目标怎么定义上。链路里任何一环跑偏，后续优化就会一起跑偏。

## 工作原理

反馈回路：

```
任务定义 → Eval Set → Grader/Judge → Reward Signal → Policy Update → New Rollouts → 任务定义
```

ORM vs PRM：

| 属性 | ORM | PRM |
|------|-----|-----|
| 打分对象 | 最终答案 | 中间步骤 |
| 信号密度 | 稀疏 | 密集 |
| 标注成本 | 低 | 高（数倍） |
| 适用场景 | 一般任务 | 数学/代码推理 |
| 失败模式 | 走捷径推理 | 高标注开销 |

Verified rewards：用程序直接验证正确性，在数学、代码、逻辑等可验证任务里替代人工偏好。

## 关键属性 / 权衡

- **Reward hacking**：钻打分系统空子，而非真正完成任务
- **Reward tampering**：直接篡改奖励计算过程
- **Alignment faking**：表面合规但隐藏不对齐意图
- **可见 CoT 局限**：模型会使用额外提示却不在 CoT 里承认；更适合当训练和监控信号，不能当完整真相
- **Constitutional AI**：把人类写的原则接进训练，用 AI feedback 替代逐条人工偏好
- **Deliberative Alignment**：把安全遵守放进推理过程，让推理能力承担安全约束
