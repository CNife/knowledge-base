# 隐藏式授权

## 定义

隐藏式（implicit grant）是 OAuth 2.0 四种授权方式中的一种，适用于纯前端应用。客户端直接从授权服务器获取令牌，不经过授权码交换步骤，令牌通过 URL 片段返回给用户代理。

## 为什么重要

隐藏式授权适用于没有后端的纯前端应用（如单页应用）。由于这类应用无法安全存储客户端密钥，授权流程被简化，令牌直接返回给前端 [S2]。

## 工作原理

隐藏式授权流程 [S2]：

1. **客户端重定向用户到授权服务器**
   - URL 包含 `client_id`、`redirect_uri`、`response_type=token`

2. **用户登录并授权**
   - 授权服务器验证用户身份
   - 用户确认授权请求

3. **直接返回令牌**
   - 授权服务器重定向回 `redirect_uri`
   - 令牌直接在 URL 片段中返回：`#access_token=xxx`
   - 前端 JavaScript 可以从 URL 中提取令牌

4. **使用令牌访问 API**
   - 前端使用令牌直接向资源服务器请求数据

## 关键属性 / 权衡

- **适用于**：纯前端应用、单页应用（SPA）
- **无需客户端密钥**：前端无法安全存储密钥
- **流程简化**：没有授权码交换步骤，直接返回令牌
- **安全风险**：令牌暴露在浏览器中，有效期应设置得很短
- **与授权码对比**：缺少后端保护，安全性较低
