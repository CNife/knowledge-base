# 系统配方

## 定义

系统配方是大模型训练中的分布式系统约束层，包括 GPU 数量、显存带宽、并行策略、容错和成本等，这些约束在训练开始前就决定了能训多大、支持多长上下文、能否跑复杂后训练。

## 为什么重要

大模型训练是分布式系统问题，而非单机深度学习问题。算力预算固定，模型大小、训练 token 量、上下文长度、serving 成本，每往一个方向多花，其他方向就得让步。大部分决定在训练开始前就锁死。

## 工作原理

关键系统决策：

```
固定算力预算 → 四个方向权衡：
  ↑ 更大模型 → GPU 内存、路由复杂度
  → 更多训练 tokens → 训练时间、数据管道成本
  ↓ 更长上下文 → Attention 成本、更小 batch size
  ← 更便宜 serving → 量化约束、更小活跃参数
```

训练稳定性问题：
- Loss spike：训练损失突增，可能需回滚 checkpoint
- 静默错误：GPU 不报错但产生错误梯度
- 通信抖动：NVLink 带宽异常、节点间通信抖动

## 关键属性 / 权衡

- **MoE**：相近计算量下扩大总参数，控住激活成本；代价是路由复杂、负载均衡难
- **muP**：超参可从小规模实验迁移到大规模训练
- **WSD learning rate**：先升后稳再衰减的学习率调度
- **长上下文约束**：128K context 改变 attention 成本、batch size、训练 curriculum
- **DeepSeek-V3 案例**：预训练 14.8T tokens，约 2.788M H800 GPU hours，无 rollback
