# 蒸馏部署

## 定义

蒸馏部署是大模型训练链路的最后阶段，包括将大模型推理能力迁移给更小 dense 模型（蒸馏）、针对特定任务的专用化、以及最终发布版本的选择。前沿模型的价值既在服务能力，也在给下一代模型提供训练数据。

## 为什么重要

发布出去的模型背后已跑完预训练、后训练、蒸馏、专用化整条链路。更强的模型持续给下一代产出训练数据。发布版本是产品决策，不是训练曲线最右边的 checkpoint。实际发布前会在多个 checkpoint 间反复比较真实任务结果、拒答风格、工具稳定性、成本和回归风险。

## 工作原理

蒸馏流程：

```
大模型 RL + verified rewards → 推理能力练出 → 推理轨迹 → 迁给更小 dense 模型
```

能力解耦解释：
- 互联网语料里知识记忆和推理能力耦合
- 预训练要求模型同时学好两件事
- 大模型先上来撑起两件事，再生成纯推理示范数据
- 小模型专注推理，不用被迫记住所有知识

部署适配：
- 很多场景不需要全能大模型，更关心成本、延迟、稳定性、可控性
- 训练终点不一定是更大，也可能是更小、更便宜、更专门

持续优化：
- Cursor Composer 2 real-time RL：Agent 能力通过生产流量持续迭代
- 训练和部署边界没消失，但反馈回路在缩短

## 关键属性 / 权衡

- **DeepSeek-R1 蒸馏**：RL 后大模型轨迹让 1.5B 到 70B dense 模型都获得明显收益
- **Llama 3.1 405B**：用于提升 8B 和 70B 的后训练质量
- **TranslateGemma**：专用模型路线，高质量数据 + 专门奖励设计
- **发布版本选择**：产品决策，不是单一指标最强点
- **前沿模型价值**：训练数据来源，不只是推理服务
