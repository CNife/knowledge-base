# 数据工程

## 定义

数据工程是将原始数据（网页、代码仓库、书籍、论坛等）经过完整漏斗处理，转化为预训练可用数据的生产工程，决定模型最终形成的能力分布。

## 为什么重要

数据工程不只是清洗数据，更是能力设计。模型看见什么、看不见什么，代码数学百科各占多大比例，这些选择直接影响模型的能力分布。去重和污染控制对结果影响很大，很多开源模型效果参差不齐，往往是数据处理质量的差距。

## 工作原理

完整漏斗处理流程：

```
原始数据源 → 文本抽取 → 语言识别 → 质量过滤 → PII 处理 → 安全过滤 → 去重 → 混合设计 → 训练分片
```

各阶段处理内容：
- **文本抽取**：从网页、PDF 等提取纯文本
- **语言识别**：识别文本语言，按语言分流
- **质量过滤**：过滤低质量内容
- **PII 处理**：隐私信息脱敏
- **安全过滤**：过滤有害内容
- **去重**：document-level 和 line-level dedup
- **混合设计**：不同类型数据的配比

## 关键属性 / 权衡

- **去重重要性**：处理重复模板、许可证文本、镜像网页、benchmark 泄漏污染
- **Data Mixing Laws**：不同类型数据占比决定能力结构
- **合成数据**：Self-Instruct、DeepSeek-R1 蒸馏轨迹、Qwen/Kimi 合成监督
- **能力迭代**：每代更强模型参与重构下一代模型的数据
