# 后训练

## 定义

后训练是预训练之后的多阶段训练流程，包括指令微调（SFT）、偏好优化（RLHF/DPO/RFT）等，决定用户真正感受到的模型差距——指令遵循、风格、拒答行为、工具使用等。

## 为什么重要

普通用户感受到的提升主要发生在预训练之后。InstructGPT 证明：1.3B 参数做过对齐的模型，在人类偏好评测里能赢过 175B 的 GPT-3。后训练改变回答方式，把怎么接任务、怎么组织输出、怎么像个配合的助手变成监督信号。

## 工作原理

主要方法对比：

| 方法 | 路径 | 特点 |
|------|------|------|
| RLHF | 模仿高质量回答 → 偏好比较做强化 | 需要奖励模型 |
| DPO | 直接从偏好对比里学 | 不需要单独训奖励模型 |
| RFT | 任务定义 + grader + 奖励信号 | 工程上更容易落地 |

DeepSeek-R1 四阶段流水线：

```
阶段1: 冷启动 SFT → 少量高质量 CoT 数据热身，收住格式和语言一致性
阶段2: 推理 RL (GRPO) → 数学/代码/逻辑可验证领域，GRPO 算法
阶段3: 拒绝采样微调 → RL 成功轨迹过滤后转成新 SFT 数据
阶段4: 对齐 RL → 有益性和安全性偏好反馈
```

## 关键属性 / 权衡

- **GRPO vs PPO**：GRPO 不需要独立价值网络，用组内排名替代绝对价值估计，工程更简洁
- **SFT 学风格**：数据长度、格式、是否带引用、是否偏好分点表达，影响输出形态
- **偏好评测偏好长回答**：容易把看起来认真的长输出当成更可靠
- **四个阶段互相依赖**：冷启动让 RL 稳定启动，RL 产生高质量数据，拒绝采样变成下一轮 SFT 输入
