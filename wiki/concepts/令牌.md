# 令牌

## 定义

令牌（token）是 OAuth 2.0 中的一种短期有效的凭证，由授权服务器颁发给客户端应用，用来代替密码访问资源服务器上的用户数据。

## 为什么重要

令牌是 OAuth 2.0 的核心凭证机制。它解决了第三方应用访问用户数据时的安全问题：用户无需向第三方透露密码，而是通过令牌授予有限、可控、可撤销的访问权限 [S1]。

## 工作原理

令牌的颁发流程 [S1]：
1. 用户按下"获取授权"按钮，请求授权
2. 屋主（用户）的手机收到对话框，显示授权请求
3. 用户确认授权
4. 门禁系统（授权服务器）向快递员（客户端）显示一个进入小区的令牌
5. 快递员向门禁系统输入令牌，进入小区（访问资源）

有了令牌以后，客户端可以向 API 请求数据，必须在 HTTP 头信息里面带上令牌：
```
Authorization: token 361507da
```

## 关键属性 / 权衡

令牌与密码的三点差异 [S1]：

- **短期有效**：令牌到期会自动失效，用户自己无法修改。密码一般长期有效，用户不修改就不会发生变化
- **可撤销**：令牌可以被数据所有者撤销，会立即失效。密码一般不允许被他人撤销
- **权限范围**：令牌有权限范围（scope），比如只能进小区的二号门，只读令牌比读写令牌更安全。密码一般是完整权限

**安全警告**：只要知道了令牌，就能进入系统。系统一般不会再次确认身份，所以令牌必须保密，泄漏令牌与泄漏密码的后果是一样的 [S1]。
